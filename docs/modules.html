API docs¶

Overview¶

At a high-level, the code is fully contained in the src/ directory and organized as illustrated below.

└── src
 ├── engine
 │   ├── step1
 │   ├── step2
 │   ├── step3
 │   ├── step4
 │   │   └── model_dev
 │   │       └── utils
 │   ├── step5
 │   └── step6
 └── ui

The engine package contains the logic that powers the tool and the ui package the Graphical User Interface bridging the engine and users.

UI¶

The modules in the engine package are organized as described below:

ui
├── detection_visualizer.py
├── __init__.py
├── layouts.py
└── ui.py

The UI class in the ui module is the entry point of the package. Detailed documentaion of the ui module can be found in src.ui package.

The layouts module contain the aspect ratio of the UI layers. The detectiion_visualizer module contains the logic to highlight detected entites in the UI.
